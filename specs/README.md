# specs/

Feature specifications for meteo-map-lab. One file per feature or decision,
written **before** implementation.

Suggested workflow:

1. `/spec-generate` — draft a spec for the feature.
2. `/spec-implement` — turn the spec into working code.
3. `/spec-finish` — mark the spec as implemented.

See [`../ai-docs/PLANNING.md`](../ai-docs/PLANNING.md) for overall architecture.
